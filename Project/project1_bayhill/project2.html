Publishing

퍼블리싱 가이드 보러가기

  • 사용기술

  •   JQuery. toggleClass( )를 이용한 햄버거 메뉴 토글
  •   Swiper.js를 이용한 스크롤 이벤트 기능
  •   Javascript fade in-out 효과
  •   Javascript 증감 함수를 이용해 숫자 카운트 버튼 구현
  • 기술서 보러가기

  • 문제 및 해결

  •   Menu 페이지 배경색

    menu 페이지 배경색은 검은색으로 설정해 뒀기 때문에 배경색이 흰 색인 서브
    페이지에서 메뉴를 열면 검은색으로 변경시키기 위해 .active 클래스를 붙여두고 Jquery toggleClass를 사용함

  •   메인 페이지 세로선 스크롤 이벤트

    메인 페이지에서 스크롤을 내릴 때 세로 선이 같이 내려오도록 구현해야 하는 부분이 있었는데 CSS after, before를 사용하여 aos 라이브러리를 적용시키려 했지만
    구현이 잘 안됐다. span 속성으로 선을 만든 후 position으로 위치를 조정하고 aos 라이브러리를 적용시켜 세로 선이 내려오도록 구현했다.

  •   모니터 해상도별 폰트 사이즈

    베이힐 풀빌라 프로젝트는 반응형 홈페이지로 만드는 것은 아니었지만, 1400px ~ 1920px 까지 문제없이 보이게 하기 위해 폰트 사이즈를 rem, px로 적용시키는 것이 아닌 vw 값으로 변환해 적용시켰다.

Review

  • 부족한 점

  • 모든 subPage의 레이아웃 구성을 똑같이 구성한 점.
        객실, 부대시설, 레스토랑 레이아웃을 다르게 구성하려고 했다면 시간은 오래     걸렸겟지만, 충분히 할 수 있었을 것 같다는 생각이 듦
  • 내부 사이트에서는 target=“_blank”를 쓰지 않는 것이 사용자의 편의성에
        적합하다는 점을 모르고 모든 서브 페이지를 새 창으로 열리게 한 점

  • 잘한 점

  • 팀원들과의 원활한 의사소통을 통해 모두 책임감 있게 프로젝트를 완료함
  • 기존 웹사이트를 최대한 완성도 높게 구현함
  • 프로젝트 기간 내 완성하여 제출함
  • 구현이 안 되는 script가 있었지만 포기하지 않고 끝까지 찾아서 구현해 냄

  • 배운 점

  • git에서는 서로 push와 pull을 했다고 소통하지 않으면 오류가 나기 때문에        커뮤니케이션의 중요성을 깨달음
  • 한번 코드가 뒤틀려진 상태로 git에 올라갔었는데 외장 하드에 이중 백업을
       하고 있었기 때문에 복구할 수 있었음. 백업의 중요성을 배움
  • 아코디언 메뉴 script 적용법과 slide script를 plugin을 사용해 적용시키는      법을 알게 됨
  • 지금까지 font-size를 rem 단위로 적용시켰었는데, vw 단위로 적용시키면
       창이 줄어들때 font-size도 같이 비율에 맞춰 줄어들게 된다는 사실을 알게 됨

  • 개인 리뷰

  • 팀 내에서 각자 자기 역할을 묵묵히 수행하고 서로 도우며 협업하는 모습이 인상적이었다. 이러한 긍정적인 분위기로 인해 의사소통도 원활해졌던 것 같다. 프로젝트를 시작하기 전 퍼블리싱 가이드라인을 작성하였지만, 평소의 코딩 습관으로 인해 가이드라인을 따르면서 코딩하는 데 시간이 더 걸렸다.
  • 코드를 깔끔하게 작성하려고 노력했지만, 자꾸만 길어져서 그 부분이 아쉬웠다. 그러나 팀원분들이 주석을 명확하게 달아주고 특이사항을 바로바로 공유해 주셔서 각자 맡은 페이지들을 취합할 때는 수월하게 진행할 수 있었다.
  • 햄버거 메뉴를 누르면 내려오고 다시 누르면 접히는 기능과 Location 페이지에 교통수단 아코디언 메뉴를 구성하는 작업은 처음부터 쉽게 구현되지 않았다. 그렇지만 팀원분들과 함께 “될 때까지 한다” 라는 마인드로 계속 노력하다 보니 마지막엔 원하는 대로 구현이 되어서 뿌듯하다.
  • 그 외 다른 팀원들이 구현한 swiper, scroll, fade in-out 등 웹페이지에서 자주 쓰이는 스크립트에 대해 많은 공부가 되었다. 실제 숙소 예약 홈페이지처럼 완벽히 구현하지는 못했지만, 숙소 홈페이지 특성상 많은 부가시설 서브 페이지를 구현했다는 점에서는 만족스러운 프로젝트였다.
